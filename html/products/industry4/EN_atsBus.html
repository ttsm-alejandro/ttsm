ATS Bus

ATS Bus is a solution that can build linkages between equipment and system in a short term at lower cost without programming. It’s one smart tool to promote IoT at factories which does not require a manual programming skill.

What ATS Bus can do

■ Collection and visualization of equipment data
■ In-process tracking of products
■ Efficient system linkage
■ Building system linkage in a short term without programming

The followings can be set with just mouse control:
  - Data definition
  - I/F・Route definition
  - Linkage definition of events and actions

Product Features

Collection and visualization of equipment data

It collects various kinds of manufacturing equipment information at manufacturing sites by setting data definition, route/event definition and actions/linkage definition, then it visualizes them on web applications.

Application example of manufacturing IoT using ATS Bus

CASE: Information gathered from manufacturing sites is sent into the database and a monitoring screen is built with a BI tool.

This case is explained in the following 3 steps:
 1. Data definition
 2. I/F and route definition
 3. Event, action, and linkage definition

【Overall structure】

0. Preparation –Building a mechanism of data collection and data disclosure

ATS Bus is a tool that builds a system linkage among nodes such as equipment, measuring devices, sensors, etc., IoT platforms, and database. As well as ATS Bus, INPUT/OUTPUT mechanism needs to be built for realizing IoT as shown in the figures below.

1. ATS Bus operation – 1. Data definition

Define identification keywords and data contents for the data collected from each equipment and measurement node, and data to send to an IoT platform and upper layer in the database, and save them with identification message.

2. ATS Bus operation – 2. I/F and route definition

Define interface, monitoring target, and transferring target for the data collected from each equipment and measurement node, and data to send to an IoT platform and upper layer in the database, and save them with identification message name.

3. ATS Bus operation – 3. Event, action, and linkage definition

Link actions such as transferring data to IoT platforms and upper layer in the database for events such as uploading data from each equipment and measurement node, and build data control between the nodes.

4. Data disclosure with BI tools

Data collected from each node at manufacturing sites are gathered into the database or sent to production system via ATS Bus. Create a report page in any style each user wants with required information and disclose it.
